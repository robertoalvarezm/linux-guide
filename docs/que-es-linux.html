De un proyecto estudiantil a millones de computadoras

Linux nació en 1991 de la curiosidad de un estudiante finlandés. Hoy corre en servidores, supercomputadoras, teléfonos Android… y en la laptop vieja que tienes en casa.

Un poco de historia

El principio: Unix y la semilla libre

En los años 70, los laboratorios Bell (AT&T) crearon Unix, un sistema operativo poderoso y elegante. Las universidades lo adoptaron, los estudiantes lo estudiaron — pero era software propietario: costoso y cerrado.

En 1983, Richard Stallman lanzó el proyecto GNU con una idea radical: construir un sistema operativo completamente libre que cualquiera pudiera usar, estudiar, modificar y compartir. Creó también la Licencia Pública General (GPL), el fundamento legal del software libre.

Foto: Wikimedia Commons

GNU tenía todo lo necesario: compiladores, editores, herramientas de desarrollo… pero le faltaba el núcleo, el “kernel” que hace que el sistema operativo funcione.

1991 — Linus Torvalds escribe el kernel

Note📬 El mensaje que cambió la computación

El 25 de agosto de 1991, Linus Torvalds publicó en un foro de internet:

“Estoy haciendo un sistema operativo libre (solo un hobby, no será grande ni profesional como GNU)…”

Ese “hobby” se convirtió en el kernel Linux — el corazón del sistema que hoy alimenta el 96% de los servidores web del mundo.

GNU tenía todas las herramientas, pero le faltaba el núcleo. Linux tenía el núcleo. Juntos formaron GNU/Linux: un sistema operativo completo, libre y gratuito.

La explosión de las distribuciones

Con el código abierto y disponible, comunidades de todo el mundo empezaron a empaquetar Linux de formas diferentes — cada una con distintos objetivos, entornos y públicos. A esas versiones se les llama distribuciones (o “distros”). Hoy existen cientos.

La filosofía detrás de Linux

Linux no es solo software: es una forma de pensar sobre quién debería controlar la tecnología.

TipLas 4 libertades del software libre (Richard Stallman)
# Libertad
0 Usar el programa para cualquier propósito
1 Estudiar cómo funciona y adaptarlo
2 Redistribuir copias
3 Mejorar el programa y publicar las mejoras

Estas libertades tienen consecuencias prácticas enormes:

  • Nadie te puede quitar el acceso por vencer una licencia.
  • Nadie recopila tus datos para venderlos.
  • Cualquier persona en el mundo puede mejorar el código — y lo hace. Miles de programadores contribuyen a Linux de forma voluntaria o dentro de empresas como Red Hat, Google, IBM o Canonical.
  • No hay fecha de caducidad arbitraria. Windows 10 dejó de recibir soporte en 2025; Linux no tiene ese problema.

Linux hoy: más presente de lo que crees

Es probable que ya uses Linux sin saberlo:

Android        → basado en el kernel Linux
Servidores web → ~96% corren Linux (Apache, Nginx…)
Supercomputadoras → top 500 del mundo: 100% Linux
Smart TVs, routers, cajeros automáticos, autos Tesla…

Lo que no domina, todavía, es el escritorio personal. Pero eso está cambiando, especialmente en educación y en contextos donde los recursos son limitados.

🏛️ ¿Por qué software libre en una universidad pública?

Una universidad pública tiene una misión particular: producir y compartir conocimiento para el bien común. Esa misión choca directamente con el modelo de software propietario.

NoteEl problema con el software propietario en lo público

Cuando una institución pública paga licencias de Windows, Microsoft Office o Adobe, está usando dinero de los contribuyentes para fortalecer el monopolio de empresas privadas —muchas veces extranjeras— sobre la infraestructura tecnológica del Estado y la educación.

El software libre resuelve esto en tres dimensiones:

💰 Sin pago de licencias

El costo de las licencias de software propietario es real y significativo:

Software Costo aproximado por equipo/año
Windows 11 Pro ~$200–$300 USD
Microsoft 365 (por usuario) ~$150 USD
Adobe Creative Cloud ~$600 USD
Total posible por máquina $900+ USD / año

Con Linux + LibreOffice + GIMP + herramientas libres: $0. Ese dinero puede destinarse a infraestructura, becas o más equipos.

🎓 Calidad de primer nivel — sin costo

El software libre no es software de segunda clase. Las herramientas que se usan en producción a nivel mundial son libres y gratuitas:

  • Sistema operativo: Linux (corre el 96% de los servidores web del mundo)
  • Suite de oficina: LibreOffice (compatible con formatos de Microsoft Office)
  • Navegador: Firefox (Fundación Mozilla, sin fines de lucro)
  • Programación: Python, R, Julia, VS Code, Git — todos gratuitos y de código abierto
  • Diseño: GIMP, Inkscape, Blender
  • Base de datos: PostgreSQL, MariaDB
  • Ciencia de datos: Jupyter, pandas, scikit-learn

Estas son exactamente las herramientas que usan universidades como MIT, Stanford y ETH Zurich. No hay versión “de paga que funciona mejor” — es el mismo software.

🔓 Transparencia y autonomía institucional

Con software libre, la universidad sabe exactamente qué corre en sus equipos. No hay cláusulas de licencia que prohíban el uso, no hay actualizaciones forzadas que rompen flujos de trabajo, no hay riesgo de que el proveedor decida subir el precio o discontinuar el producto.

La institución tiene el control — no la empresa.

🌍 Gobiernos que ya migraron — o están migrando — a Linux

Si esto te parece una idea experimental de entusiastas, aquí está la realidad: gobiernos de países desarrollados están abandonando Windows activamente, por razones de soberanía digital, seguridad y costo.

🇫🇷 Francia (2026) — La decisión más reciente

Francia anunció planes para migrar parte de sus computadoras gubernamentales de Windows a Linux, buscando reducir su dependencia de la tecnología estadounidense. El ministro David Amiel declaró que el esfuerzo busca “recuperar el control de nuestro destino digital” y que el gobierno ya no puede aceptar no tener control sobre sus datos e infraestructura digital.

La migración comenzará con los equipos de la agencia digital del gobierno francés, DINUM. Esto no es un caso aislado: Francia opera más de 103,000 computadoras con GendBuntu, una distribución personalizada basada en Ubuntu utilizada por la Gendarmería Nacional.

🇩🇪 Alemania — Schleswig-Holstein

En abril de 2024, el estado alemán de Schleswig-Holstein anunció la migración completa de más de 30,000 trabajadores gubernamentales de Windows a Linux, incluyendo LibreOffice, Nextcloud y Thunderbird — una de las migraciones más grandes en la historia de los gobiernos europeos.

La motivación va más allá del ahorro económico: al alojar herramientas de código abierto en servidores propios dentro de la UE, se garantiza que la información gubernamental no fluya hacia jurisdicciones extranjeras.

🇩🇰 Dinamarca

El Ministerio de Asuntos Digitales de Dinamarca se comprometió a una transición completa a Linux y LibreOffice, con el objetivo de eliminar la dependencia del software de Microsoft.

Y más…

India desplegó Maya OS, un sistema basado en Ubuntu desarrollado por su organización de defensa (DRDO), para reemplazar Windows en instalaciones militares. Hay migraciones parciales o sectoriales también en Rusia, China, Brasil y Turquía.

Important¿Por qué ahora?

Esta tendencia se enmarca en la creciente inestabilidad e impredecibilidad geopolítica, y en la conciencia de los legisladores europeos sobre su excesiva dependencia de la tecnología estadounidense. Los gobiernos están descubriendo lo que el software libre lleva décadas argumentando: depender de un proveedor privado extranjero para tu infraestructura digital es un riesgo estratégico.

Para una universidad pública, el argumento es exactamente el mismo — solo que a escala institucional.

¿Por qué instalar Linux en laptops viejas?

Aquí es donde la filosofía se vuelve práctica. Rescatar una laptop vieja con Linux no es solo una solución técnica — tiene implicaciones sociales y ambientales reales.

♻️ Menos basura electrónica

Los desechos electrónicos (e-waste) son el tipo de residuo de más rápido crecimiento en el mundo. Una laptop contiene metales como plomo, mercurio y cadmio que contaminan suelos y agua cuando terminan en basureros.

Instalar Linux puede extender la vida útil de una laptop entre 5 y 10 años adicionales. Una máquina que “ya no sirve para Windows 11” puede correr Linux perfectamente durante años más.

Hay laptops del año 2010–2015 que con una distro ligera como Lubuntu o Xubuntu:

  • Navegan en internet sin problemas.
  • Reproducen video en HD.
  • Corren herramientas de oficina, programación y diseño básico.
  • Sirven perfectamente para estudiar, escribir y aprender.

💻 Equipos para estudiantes de bajos recursos

No todos los estudiantes tienen acceso a una laptop nueva. Una computadora vieja rescatada con Linux puede ser la diferencia entre tener o no tener herramientas digitales para estudiar, hacer tareas o aprender a programar.

El costo de esta solución es prácticamente cero:

Concepto Costo
Linux (cualquier distro) Gratis
LibreOffice (suite de oficina) Gratis
Navegador web (Firefox) Gratis
Editor de código (VS Code, Gedit) Gratis
Laptop vieja de segunda mano $300–$800 MXN

Con una inversión mínima (o ninguna, si la laptop ya existe), se puede equipar a un estudiante con todas las herramientas que necesita.

🎒 Un banco de laptops prestables

La idea es simple y ya funciona en muchas escuelas y comunidades:

Note🏫 Cómo funciona un banco de laptops con Linux
  1. Recolectar laptops viejas donadas por familias, empresas o instituciones.
  2. Evaluar hardware: revisar que pantalla, teclado, batería y disco funcionen mínimamente.
  3. Instalar una distro ligera (Lubuntu, Xubuntu o Mint XFCE).
  4. Configurar con las aplicaciones básicas: navegador, suite de oficina, editor de texto, herramientas de programación.
  5. Prestar la laptop a estudiantes que la necesiten durante el semestre.
  6. Recuperar, limpiar y reutilizar al terminar el periodo.

Linux facilita esto de forma especial: como no hay licencias que administrar, puedes clonar la misma configuración en decenas de laptops sin costo adicional. Herramientas como Clonezilla permiten copiar la instalación completa de una laptop a otra en minutos.
